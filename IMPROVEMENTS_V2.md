# 학습 개선 사항 (v2)

## 🎯 개선 목표
학습된 모델이 선공일 때만 이기는 문제 해결

## 📊 문제 분석
- **증상**: Final Model vs Test Model → 선공 100% 승리
- **원인**: AI가 공격적인 초반 전략만 학습, 방어/후반 전략 부재
- **근본 원인**: 
  1. 승/패 보상(±1.0)만 있어 중간 과정 학습 불가
  2. Opponent 업데이트가 너무 잦아 불안정 (5,000 스텝)
  3. 학습 스텝이 부족 (100,000)

## ✨ 개선 사항

### 1️⃣ 보상 체계 개선 (`hearthstone_env.py`)

**기존:**
```python
# 승리: +1.0, 패배: -1.0만 존재
```

**개선:**
```python
# 최종 목표
승리: +1.0
패배: -1.0

# 중간 보상 (매 스텝마다)
체력 차이: ±0.001 per HP      # 생존력 학습
필드 우위: ±0.002 per 공격력  # 전략적 우위 학습
카드 우위: ±0.001 per 카드    # 자원 관리 학습

총 중간 보상: -0.1 ~ +0.1 (클리핑)
```

**효과:**
- ✅ 체력 관리 학습 (무조건 공격 X)
- ✅ 필드 컨트롤 학습 (미니언 교환 전략)
- ✅ 자원 관리 학습 (카드 드로우 중요성)

### 2️⃣ 하이퍼파라미터 튜닝 (`train_selfplay.py`)

| 파라미터 | 기존 | 개선 | 이유 |
|---------|------|------|------|
| **Learning Rate** | 3e-4 | 1e-4 | 더 안정적인 수렴 |
| **N Steps** | 2048 | 4096 | 더 많은 경험 수집 |
| **Batch Size** | 64 | 128 | 안정적인 그래디언트 |
| **Opponent Update** | 5,000 | 10,000 | 안정적인 Self-Play |
| **Total Steps** | 100,000 | 200,000 | 충분한 학습 시간 |
| **Save Interval** | 10,000 | 20,000 | 저장 빈도 조정 |

**효과:**
- ✅ 학습 안정성 향상 (폭발적 그래디언트 방지)
- ✅ Self-Play 안정성 (opponent가 너무 자주 안 바뀜)
- ✅ 2배 긴 학습으로 전략 발달 시간 확보

### 3️⃣ 예상 학습 시간

**이전:**
- 100,000 스텝 / 198 fps = ~8.5분

**현재:**
- 200,000 스텝 / 198 fps = ~17분
- n_steps 증가 (2048→4096): 업데이트 횟수 절반
- 실제 예상 시간: **약 15-20분**

## 🚀 학습 실행

```bash
python train_selfplay.py
```

## 📈 예상 결과

**개선 전:**
- 선공 승률: 100%
- 후공 승률: 0%
- 전략: 무조건 공격

**개선 후 목표:**
- 선공 승률: 60-70%
- 후공 승률: 30-40%
- 전략: 체력 관리, 필드 컨트롤, 자원 관리

## 🔧 추가 개선 가능성

만약 여전히 문제가 있다면:

1. **보상 가중치 조정**
   - 체력/필드/카드 보상 비율 변경
   
2. **커리큘럼 학습**
   - 초반: 랜덤 상대 (기본 학습)
   - 중반: Self-Play (전략 발달)
   - 후반: 강한 고정 상대 (일반화)

3. **탐험 강화**
   - ent_coef 증가 (0.01 → 0.02)
   - 다양한 전략 시도

4. **더 긴 학습**
   - 500,000 ~ 1,000,000 스텝
